# 前言

在前一段时间，借助强大的 DeepSeek 的力量实现了成长城堡部分数据的查询 API，全部用 Python 完成的 (需要说明一下，DeepSeek 只是帮助我完成了一些无关紧要的杂活，核心算法以及数据库的设计还是由我亲自完成的)。它能够查询波速信息，当然目前主要也只做了这个功能。碍于一些原因，目前源程序只留着自用。现在开放的 API 有两个 URL：

```
https://fnxag.eu.org/gcapi
```

和

```
https://xane.eu.org/gcapi
```

前者是经过 Cloudflare 代理的，而后者是直接访问源服务器。两者的内容完全没有差别，只是对于国内的访问速度有所差异。因此，对于国内的用户，更建议使用**第一个** URL。

# API 说明

按道理来说，这个 API 需要和我这个项目配套：

::github{repo="FnxAg/GrowCastleCalculator"}

我在里面实现了对相关字段的解析，并且简单配置好了就能用。

## URL 和 Request

以 `https://fnxag.eu.org/gcapi` 为例，发起 `GET` 类型请求，后面接固定字段 `/season`，附带相关参数，如下：

```
https://fnxag.eu.org/gcapi/season/{时间}/{对象类型}/{对象名称}
```

| 字段名称 | 可选对象               | 说明                                                                                         |
| -------- | ---------------------- | -------------------------------------------------------------------------------------------- |
| 时间     | `now` / `last` / `all` | `now`：当前赛季<br />`last`：上赛季<br />`all`：数据库中的所有对应条目，取决于服务器的数据量 |
| 对象类型 | `players` / `guilds`   | `players`：玩家<br />`guilds`：公会                                                          |
| 对象名称 | 名称                   | 根据需要查询的对象，填写玩家或公会的游戏名称                                                 |

> \[!NOTE] 提示
>
> 如果玩家一直使用默认名称，如 `playerxxxxxxxx`，将无法查询。

## Response

以对玩家的请求为例，服务器会返回对应的 JSON 格式数据，大致格式如下：

```json
[
	{
		"id": 1, 
		"name": "aaa", 
		"wave": 1234567, 
		"score": 20000, 
		"wph": 100, 
		"season": "20260709", 
		"last_fetch": 1783000000
	},
	{
		"...": "..."
	},
]
```

在 Python 中，这是一个 `list[dict]` 类型，下面对每个字段进行说明：

| 字段         | 类型                | 说明                                      |
| ------------ | ------------------- | ----------------------------------------- |
| `id`         | `int`               | 数据库 id，可忽略                         |
| `name`       | `string`            | 游戏名称                                  |
| `wave`       | `int`               | 总波                                      |
| `score`      | `int`               | 赛季波                                    |
| `wph`        | `int`               | 小时内波数 / 波速                         |
| `season`     | `string`            | 赛季代号，以赛季开始的 UTC 时间年月日设置 |
| `last_fetch` | `int` / `timestamp` | 服务器抓取数据的时间戳，可忽略            |

如果 `wph` 计算时，没有上一次的记录，或正好处在赛季交替的前后，其会为 `null`。

需要说明的是，在上面的成长城堡计算器中，请求时间为 `all`，并且对响应的处理，只解析了 `wph` 和 `season` 字段，其他的目前都未使用，可能未来也不会用到。

# 其他内容

除了这种 API，还有一个实时更新的榜单，在这个地方：

```
https://fnxag.eu.org/down_bot
```

这是一个使用 [ntfy](https://ntfy.sh) 实现的通知服务器，我也把它拿来作为一个用于查看的工具了。只要它还在继续更新，那就说明还能继续用。
